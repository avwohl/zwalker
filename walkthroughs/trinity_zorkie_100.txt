# Trinity is V4: there is no status-line convention, so the standard V3
# score/turns globals (17/18) hold ordinary game data. The true score and
# move counters live in the game's own SCORE (variable 170) and MOVES
# (variable 149) globals - zwalker's get_score()/get_turns() know this via
# the per-build override table in zmachine.py, verified against the game's
# own SCORE verb output at multiple checkpoints.
#
# Why a pinned seed: two endgame facts are rolled fresh each game and this
# recording carries seed 1's values. (1) The jeep radio dial is set to a
# random frequency - seed 1 reads 32, hence "set slider to 32" to tune the
# walkie-talkie. (2) The wire-colour legend on the cardboard bookmark is a
# random permutation - seed 1 scribbles "RD=INF BL=GND ST=POS WH=DET", and
# the breaker test names "the positive line on X" over the walkie-talkie,
# hence "cut striped wire" for the finale. Ambient PROB events (magpie
# chatter, skink squirms, bee/crab/dolphin flavor text, roadrunner
# comings and goings) consume RNG draws that the recording's fixed pacing
# absorbs; the big set-pieces (tide/coconut clock, Styx dory schedule,
# island PA countdown, auto-sequencer at Zero minus 45 seconds) are
# turn-clock driven and deterministic.
#
# Note on spelling: movement commands are written as full direction words
# with a trailing period ("northeast."), which Trinity's parser treats
# exactly like the bare direction. The period form is used because Trinity
# (V4) exposes no location global for the harness's room-change detection,
# and several room descriptions ("The white door in its stem is closed.")
# would otherwise trip the replay pathway's blocked-move rollback.
